○ DFS 서비스의 클라이언트 쪽 미들웨어 기능은 (1) 클라이언트 응용에 라이브러리 형태로 응용에 포함할 수도 있고 (2) 별도의 service (daemon) process로 실행될 수도 있음. 이 외에도 다른 구현도 가능하며, 방법은 각자 선택함 
○ DFS 서비스는 stateful file service로 해도 되고 stateless file service 로 구현해도 됨. 각자 선택해 구현함 
○ Client 컴퓨터가 아닌 다른 컴퓨터에 file server가 실행되게 함. 따라서 문제 1,2는 두 대의 컴퓨터를 이용하고 문제 3은 3대를 이용함. 

○ 파일 서버의 디스크에는 임의의 내용을 가진 1~5 KB 정도 크기의 샘플 텍스트 파일들을 5개 만들어 놓음. 이 파일 중 하나는 testdata.txt 라는 이름의 파일이도록 준비함

① remote file server의 디렉토리 내용을 읽어와 local file 
○ DFS 서비스의 클라이언트 쪽 미들웨어 기능은 (1) 서버와의 통신, 즉 통신 미들웨어 기능 (2) 파일 접근을 위한 access transparency(응용으로 하여금 remote 자원을 local 자원과 같은 방법으로 액세스하게 만들어 주는 기능)과 location transparency(응용으로 하여금 파일 서버의 물리적인 주소를 몰라도 접근할 수 있게 하는 기능)을 제공함
   - Location transparency를 위해 간단하게 라도 name service 를 이용해야 함. 그러나 이 기능은 문제 3에서만 필요로 하며, 문제 1번에서는 name service 를 이용하지는 않아도 됨. 클라이언트 미들웨어 설정 값으로 미리 primary file server의 물리적 주소값을 알고 있게 구현함
○ 문제 1용 서버측 미들웨어는 (1) 통신 미들웨어 기능 (2) 클라이언트의 파일 액세스 요청을 파일 서버 응용으로 전달 (3) 파일 서버와 클라이언트의 서비스 세션 관리(stateful file service 의 경우)를 수행함
○ 문제 2용  서버측 미들웨어는 (1) 통신 미들웨어 기능 (2) 파일 서버 스토리지에 들은 파일들을 모니터링하고 (3) 필요시 서버들끼리의 파일 동기화하는 기능을 담당함s
